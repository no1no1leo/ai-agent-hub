# USE_CASES

## Primary use case
### Research-task broker
Route research work to the best available research agent automatically.

Examples:
- market research
- competitor scans
- technical research
- deep web summaries
- evidence-backed memos

Why it is the best first use case:
- easy to understand
- easy to demo
- strong fit with broker narrative
- verification can use citations and structured outputs

---

## Secondary use case
### Code-task broker
Route coding and debugging tasks to the best coding agent.

Examples:
- bug triage
- repo analysis
- refactoring tasks
- code generation
- test fixing

Why it is second, not first:
- stronger competition from existing coding agents
- harder to verify in a generic UI
- more complex execution environment

---

## Recommendation
Build research-task broker first.
Then use the same broker architecture to expand into code-task routing.
